# Исправление GitHub Pages для JuristAI v2

Если вы видите ошибку 404 на https://k09-0.github.io/juristai-v2, это значит, что GitHub Pages не настроена или репозиторий не загружен правильно.

## 🔧 Решение (5 минут)

### Шаг 1: Проверить репозиторий на GitHub

1. Открыть https://github.com/K09-0/juristai-v2
2. Если репозитория нет - нужно его создать и загрузить файлы
3. Если репозиторий есть - перейти к Шагу 2

### Шаг 2: Проверить наличие папки `/docs`

1. На странице репозитория должна быть папка `docs`
2. Внутри папки `docs` должен быть файл `index.html`
3. Если папки нет - нужно загрузить файлы из папки `/docs` в репозиторий

### Шаг 3: Включить GitHub Pages

1. Открыть репозиторий https://github.com/K09-0/juristai-v2
2. Нажать на **Settings** (в верхнем меню)
3. В левом меню выбрать **Pages**
4. Под **"Source"** выбрать:
   - **Branch**: `main`
   - **Folder**: `/docs`
5. Нажать **Save**
6. Подождать 1-2 минуты
7. Страница должна быть доступна по адресу: https://k09-0.github.io/juristai-v2

### Шаг 4: Проверить CNAME (для пользовательского домена)

Если хотите использовать домен `juristai.site`:

1. В Settings → Pages найти поле **Custom domain**
2. Ввести `juristai.site`
3. Нажать **Save**
4. GitHub создаст файл `CNAME` в репозитории
5. Перейти в настройки DNS вашего домена (domainnameapi.com)
6. Добавить CNAME запись:
   - **Name**: `@` или оставить пусто
   - **Value**: `k09-0.github.io`
7. Подождать 5-10 минут для распространения DNS

## 🚀 Если все еще не работает

### Проблема 1: Папка `/docs` пуста или отсутствует

**Решение:**
```bash
# На вашем компьютере в папке проекта
cd juristai-v2

# Скопировать собранные файлы в docs
cp -r client/dist/* docs/

# Или если вы на Windows, используйте File Explorer:
# 1. Откройте папку client/dist
# 2. Скопируйте все файлы
# 3. Вставьте в папку docs
# 4. Замените существующие файлы
```

### Проблема 2: GitHub Pages не обновляется

**Решение:**
1. Откройте Settings → Pages
2. Измените Source на другую ветку, потом вернитесь на `main`
3. Нажмите Save
4. Подождите 2-3 минуты

### Проблема 3: Все еще 404

**Решение:**
1. Проверьте, что в папке `/docs` есть файл `index.html`
2. Откройте Settings → Pages и проверьте, что выбрана папка `/docs`
3. Попробуйте очистить кэш браузера (Ctrl+Shift+Delete)
4. Попробуйте открыть в приватном окне браузера
